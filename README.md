# **คำถามก่อนทำแลป**

**1. จะใช้ประเภทที่ดินกี่ Class? — นิยาม**

ในงานนี้เลือกใช้การจำแนกประเภทที่ดินจำนวน 5 ประเภท (5 Classes) ได้แก่

- Forest (0): พื้นที่ป่าไม้หรือพืชพรรณหนาแน่น มีค่า NDVI สูง
- Water (1): แหล่งน้ำ เช่น แม่น้ำ คลอง บึง มีค่า NDWI สูง
- Building (2): พื้นที่สิ่งปลูกสร้างหรือเขตเมือง มีการสะท้อนใน band สูงโดยเฉพาะในช่วง visible และ SWIR
- Bareland (3): พื้นที่ดินเปล่า ไม่มีพืชปกคลุม เช่น พื้นที่ก่อสร้างหรือดินโล่ง
- Agriculture (4): พื้นที่เกษตรกรรม มีพืชแต่ไม่หนาแน่นเท่าป่า และมีการเปลี่ยนแปลงตามฤดูกาล

**2. จะหา Training Samples อย่างไร?**

การสร้าง Training Samples ใช้วิธี วาด Polygon เอง (Manual Digitization) บนภาพดาวเทียมใน Google Earth Engine โดยอ้างอิงจากลักษณะพื้นที่จริง เช่น สี ลักษณะ texture และตำแหน่งเชิงภูมิศาสตร์

วิธีนี้ช่วยให้สามารถควบคุมคุณภาพของข้อมูลได้ดี และเหมาะสำหรับการเรียนรู้พื้นฐานของการจำแนกข้อมูลเชิงพื้นที่ อย่างไรก็ตามอาจมีข้อจำกัดเรื่องจำนวนข้อมูลและความเป็นตัวแทนของพื้นที่ทั้งหมด

**3. การแบ่ง Train/Validation**

ในงานนี้ใช้วิธีแบ่งข้อมูลแบบ Random Split 80/20 โดยสุ่มแบ่งข้อมูลเป็นชุดฝึก (Training) 80% และชุดทดสอบ (Validation) 20%

อย่างไรก็ตาม วิธีนี้มีข้อจำกัดเนื่องจากข้อมูลเชิงพื้นที่มักมี Spatial Autocorrelation ทำให้ pixel ที่อยู่ใกล้กันมีค่าคล้ายกัน ส่งผลให้ข้อมูล train และ test มีลักษณะใกล้เคียงกัน และอาจทำให้ค่าความแม่นยำสูงเกินจริง

ในทางตรงกันข้าม วิธี Spatial Cross-validation จะช่วยลดปัญหานี้โดยการแบ่งข้อมูลตามพื้นที่ ทำให้การประเมินผลมีความสมจริงมากขึ้น แต่มีความซับซ้อนในการดำเนินการ

**4. Feature ที่ใช้ — เหตุผล**

ในการจำแนกใช้ข้อมูลทั้งหมด 9 Features ได้แก่

Band: B2, B3, B4, B8, B11, B12
Index: NDVI, NDMI, NDWI

โดย Band ต่าง ๆ ช่วยสะท้อนลักษณะทางกายภาพของพื้นผิว เช่น สี ความชื้น และวัสดุ
ขณะที่ Index เช่น NDVI ใช้แยกพืชพรรณ, NDWI ใช้แยกน้ำ, และ NDMI ใช้บอกความชื้น

การใช้ทั้ง Band และ Index ร่วมกันช่วยเพิ่มความสามารถของโมเดลในการแยกประเภทที่ดินได้อย่างแม่นยำมากขึ้น


# **คำถามสำหรับใช้คิดและตอบใน README**

**1. ถ้าเพิ่ม Training Samples อีก 2 เท่า ความแม่นยำจะเพิ่มขึ้นเท่าไหร่?**

จากการทดลองพบว่า เมื่อเพิ่มจำนวน Training Samples จากประมาณ 50 ตัวอย่างต่อ class เป็นประมาณ 100 ตัวอย่างต่อ class ความแม่นยำของโมเดลเพิ่มขึ้นเพียงเล็กน้อย ไม่ได้เพิ่มขึ้นอย่างมีนัยสำคัญ

สาเหตุเนื่องจากโมเดลสามารถเรียนรู้ pattern หลักของข้อมูลได้ตั้งแต่ sample ขนาดเล็กแล้ว โดยเฉพาะเมื่อใช้ Random Forest ซึ่งมีความสามารถในการ generalize ได้ดี

อย่างไรก็ตาม การเพิ่มจำนวนข้อมูลช่วยให้ผลลัพธ์มีความเสถียรมากขึ้น และลดความผันผวนของค่าความแม่นยำ 

**2. Spatial Autocorrelation มีผลต่อความแม่นยำอย่างไร?**

Spatial autocorrelation ในข้อมูล training ทำให้ค่าความแม่นยำที่รายงานสูงกว่าความเป็นจริง เนื่องจาก pixel ที่อยู่ใกล้กันมักมีค่าคล้ายกัน ทำให้ข้อมูล train และ test มีลักษณะซ้ำกัน

ส่งผลให้โมเดลดูเหมือนมี performance สูง แต่จริง ๆ แล้วอาจไม่สามารถ generalize ไปยังพื้นที่ใหม่ได้ดี

ดังนั้น การใช้ random split แบบ 80/20 อาจทำให้เกิด overestimation ของ accuracy และควรใช้ spatial cross-validation เพื่อลด bias นี้

**3. Class ใดที่โมเดลทำได้แย่ที่สุด และแก้ยังไง?**

จากผลการทดลองพบว่า class ที่โมเดลทำได้แย่ที่สุดคือ bareland และ building รวมถึงบางกรณี forest และ agriculture เนื่องจากมีลักษณะ spectral ใกล้เคียงกัน

ปัญหานี้เกิดจากการที่ค่าการสะท้อนแสงในช่วง band ต่าง ๆ เช่น SWIR และ NIR มีความใกล้เคียงกันในบางพื้นที่

วิธีแก้ไข ได้แก่

เพิ่ม training samples ใน class ที่สับสน, เพิ่ม feature เช่น texture หรือ index เพิ่มเติม, ใช้ spatial sampling เพื่อลด bias, ทดลองใช้ model ที่ซับซ้อนขึ้น เช่น Gradient Boosting

**4. ถ้าทำซ้ำในพื้นที่อื่น ต้องเปลี่ยนอะไร / ใช้อะไรซ้ำได้?**

หากต้องทำซ้ำในพื้นที่อื่น จำเป็นต้องเปลี่ยนขอบเขตพื้นที่ศึกษา (ROI) และสร้าง training samples ใหม่ให้เหมาะสมกับลักษณะภูมิประเทศของพื้นที่นั้น

นอกจากนี้ อาจต้องปรับ class definition ให้สอดคล้องกับสภาพพื้นที่ เช่น เพิ่มหรือลดประเภทที่ดิน

อย่างไรก็ตาม ส่วนของ workflow เช่น การเตรียมข้อมูล การคำนวณ band และ index การใช้โมเดล Random Forest และการประเมินผล สามารถนำไปใช้ซ้ำได้โดยไม่ต้องเปลี่ยนแปลง

**5. วิธีที่เลือก / เหตุผล / ข้อจำกัด / สิ่งที่ค้นพบ**

ในการทดลองนี้เลือกใช้วิธีการสร้าง Training Data แบบ manual และใช้ Random Forest เป็นโมเดลหลัก เนื่องจากสามารถจัดการข้อมูลหลายมิติได้ดีและไม่ต้องปรับ parameter มาก

เหตุผลที่เลือกวิธีนี้คือเหมาะสมกับข้อมูลขนาดเล็ก (≈50 samples ต่อ class) และสามารถเรียนรู้ pattern ได้อย่างมีประสิทธิภาพ

อย่างไรก็ตาม ข้อจำกัดที่พบคือจำนวนข้อมูลยังมีน้อย ทำให้โมเดลอาจยังไม่สามารถ generalize ได้ดี และค่าความแม่นยำอาจได้รับผลกระทบจาก spatial autocorrelation

จากการทดลองพบว่า Feature อย่าง NDVI และ band NIR (B8) มีบทบาทสำคัญในการจำแนกประเภทพืชพรรณ ขณะที่ NDWI มีความสำคัญในการแยกแหล่งน้ำออกจากพื้นที่อื่น
